飞书集成、Token 优化、记忆管理策略

1 配置飞书 (Feishu) 机器人

将 OpenClaw 接入飞书，让你在企业IM中直接使用 AI 助手。支持群聊、私聊、@提及等多种交互方式。

1.1 创建飞书应用

  1. 1. 访问 飞书开放平台
  2. 2. 点击"创建企业自建应用"
  3. 3. 填写应用名称 (如：OpenClaw 助手)
  4. 4. 在"凭证与基础信息"中记录 App ID 和 App Secret

1.2 配置机器人权限

# 需要添加的权限

消息：发送消息、读取消息
群组：获取群组信息
事件订阅：接收消息事件

1.3 配置 OpenClaw 飞书插件

在 workspace 中创建或编辑飞书配置文件：

# ~/.openclaw/workspace/feishu-config.json
{
"appId": "cli_a1b2c3d4e5f6",
"appSecret": "xxxxxxxxxxxxxxxx",
"verificationToken": "your_verification_token",
"encryptKey": "your_encrypt_key"
}

💡 提示

配置完成后，在飞书开放平台配置事件订阅 URL，格式为： https://your-domain.com/feishu/event

2 Cache-TTL 上下文缓存修剪

长时间对话会积累大量上下文，导致 Token 消耗激增。Cache-TTL (Time-To-Live) 机制可以自动修剪过期缓存，节省成本。

工作原理

  • • 每条消息缓存都有时间戳
  • • 超过设定 TTL 的缓存会被标记为可清理
  • • 当上下文接近 token 限制时，优先清理过期缓存

配置示例

# ~/.openclaw/workspace/config.json
{
"cache": {
"ttl": 3600, // 缓存有效期 (秒)，默认 1 小时
"maxSize": 100, // 最大缓存条数
"cleanupInterval": 300 // 清理间隔 (秒)
}
}

⚠️ 注意

TTL 设置过短可能导致重要上下文丢失，建议根据实际场景调整：
• 短时任务：300-600 秒
• 日常对话：1800-3600 秒
• 长期项目：7200+ 秒

3 长文本压缩 (Auto-compaction)

当对话历史过长时，自动将早期对话压缩为摘要，保留核心信息的同时大幅减少 token 占用。

配置压缩策略

# ~/.openclaw/workspace/config.json
{
"compaction": {
"enabled": true,
"threshold": 50, // 消息数超过 50 条触发压缩
"targetSize": 20, // 压缩后保留 20 条
"summaryModel": "qwen2.5:7b" // 用于生成摘要的模型
}
}

压缩效果示例

压缩前 (50 条消息):

完整的 50 轮对话历史，约 15000 tokens

压缩后 (20 条 + 摘要):

"[前 30 轮讨论了项目架构设计，确定了使用 Node.js + TypeScript 技术栈...]" + 最近 20 条消息，约 5000 tokens

4 记忆落盘 (Memory Flush)

将短期工作记忆定期写入磁盘，确保 Agent 重启后不丢失重要上下文。支持自动和手动两种模式。

自动落盘配置

# ~/.openclaw/workspace/config.json
{
"memory": {
"autoFlush": true,
"flushInterval": 600, // 每 10 分钟自动落盘
"flushOnExit": true, // 退出时强制落盘
"memoryDir": "./memory" // 记忆文件存储目录
}
}

手动落盘命令

# 在聊天中发送
/memory flush

# 查看当前记忆状态
/memory status

✅ 最佳实践

  • • 重要任务开始前手动执行 /memory flush
  • • 长时间运行的 Agent 启用自动落盘
  • • 定期清理过期的 memory 文件释放空间
